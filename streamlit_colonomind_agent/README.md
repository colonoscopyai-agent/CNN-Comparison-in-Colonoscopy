A Streamlit app that classifies a colonoscopy image into the **Mayo Endoscopic
Subscore** classes `MES0–MES3` using one of five trained hybrid models
(ResNet-50, DenseNet-121, EfficientNet-B4, ConvNeXt-Tiny, ViT-B/16).

Each model is a **3-input hybrid**: the CNN/ViT image branch is fused with
20 handcrafted texture features (17 wavelet + 3 GLCM) and a 2-D UMAP
projection of those features.

## App layout

1. **Upload image** – drop in a colonoscopy frame.
2. **Model performance report** – accuracy, precision, recall/sensitivity,
   specificity, F1, QWK and the confusion matrix for the model you picked.
3. **Run diagnosis & result** – the image plus the predicted class,
   confidence and per-class probabilities.
4. **Q&A agent** – ask about the metrics, the MES classes, or the last
   prediction. Answers are grounded in `deployment_report.json`; an optional
   sidebar toggle routes questions to Claude (needs `anthropic` + an API key).

The **model (weight) dropdown** lives in the sidebar.

## Run locally

```bash
cd streamlit_colonomind_agent
pip install -r requirements.txt
streamlit run app.py
```

## Deploy to Streamlit Community Cloud

1. Push this folder to a GitHub repo (see the note on weights below).
2. On https://share.streamlit.io → **New app**, pick the repo/branch and set
   **Main file path** to `streamlit_colonomind_agent/app.py`.
3. Community Cloud installs `requirements.txt` automatically. No `packages.txt`
   is needed because we use `opencv-python-headless`.
4. First load is slow: TensorFlow is large and the ViT model downloads
   `google/vit-base-patch16-224-in21k` from Hugging Face on first use.

### ⚠️ Model weights are large

The five `.weights.h5` files (EfficientNet-B4 and ViT in particular) can total
several hundred MB. GitHub blocks single files >100 MB and Community Cloud has
limited repo size, so **don't commit raw weights** to a normal repo. Options:

- Track `weights/*.h5` with **Git LFS**, or
- Host the weights externally (Hugging Face Hub, S3, Google Drive, a GitHub
  Release) and download them into `weights/` at app startup.

### Version pinning matters

`preprocess_artifacts.joblib` (StandardScaler, UMAP, LabelEncoder) and the
`.weights.h5` files were serialized in your **training** environment. Pin
`tensorflow`, `scikit-learn`, `umap-learn` and `numpy` in `requirements.txt`
to the versions you trained with, otherwise you may hit unpickling warnings or
weight-loading errors.
